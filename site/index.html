주안점

1) 분석, 설계 (ux 재설계) 방안

  • 2014년 시스템 구축된 이후에 지속적인 유지보수를 진행하면서 기능의 추가, 변경 사항을 반영하였으나, 그때그때 반영하다보니 전체적인 기능이 구조화가 되지 않은 상태로 운영되었고, 유지보수로 개발하는데 한계와 관련된 설계산출물에 대한 반영도 미흡했음
  • 1차년도에서는 사면통합정보 관련기능 개발과 UX 관점에서 레벨1의 상위수준에서 고도화작업을 수행
  • 2차년도에서는 하위레벨 수준까지 전체 기능의 구조화를 위한 재설계가 적정하게 이루어지고, ux 관점에서 불편사항과 기능개선이 필요한 사항들을 반영하여 개발되었는지 확인

2) 산사태정보서비스 홈페이지 개편 적정성

  • 현재 산사태정보서비스의 대국민서비스를 별도 페이지로 서비스하지 않고 업무서비스와 동일한 페이지에서 서비스되고 있으며, 이번 2차년도서는 별도 페이지로 만들도록 하고 있음
  • 대국민서비스의 경우 표준이나 접근성을 중요시하고 있음
  • 실제 현재 운영되고 있는 산사태정보시스템의 웹표준과 웹접근성 점검을 점검한 결과 html오류/css오류/웹접근성 오류가 다수 발생하고 있는 것을 확인
  • 별도 페이지로 새롭게 개발되는 대국민서비스에선 웹표준, 웹접근성 등이 충분히 반영되어 대국민서비스가 제공되는지 점검

3) 산사태 예측정보 위험지도 점검

  • 산사태 예측과 관련한 정보는 토양함수지수로 주의보를 발령하는 부분과 산사태 위험(지)도를 작성해서 제공하는 부분으로 구성
  • 산사태 예측정보는 KLES모델이라 불리는 산사태 토양함수지수를 토대로 주의보와 경보를 발령
  • 기상청 강우분포와 지질특성을 전국 11개 권역으로 구분하여 토양내 빗물의 양을 토양함수지수로 산정하고 주의보 발령
    • 주의보 : 권역별 토양함수지수 80%도달 시
    • 경보 : 권역별 토양함수지수 100%도달 시
  • 산사태 위험(지)도 : 산사태 발생인자 중 영향이 큰 간접요인 9개인자(임상, 경급, 사면경사 등)와 인위적 변화지 보정인자를 적용하여 1 ~ 5등급으로 구분하여 제공하는 지도
  • 실시간 산사태위험도 : 산사태위험지도에 실시간 강우상황을 반영하여 행정구역별 또는 유역별로 제공하는 지도
  • 산사태 위험지 판정식에 따라 셀단위 5단계 위험도 등급 표시(유역/동리/읍면동/수군구 단위 4단게 위험도 등급 표시)
  • 산사태 위험지 판정식은 과거 산사태 발생한 지역의 각 인자를 독립변수로 놓고, 산사태 발생유무를 종속변수로 구성해서 확률값으로 예측하는 로지스틱회귀모형을 적용해서 분류
  • 산사태발생 예측의 정확도 향상이 rfp에 언급되어 있고, 기존 산사태정보서비스를 개편하는 것이 본사업의 주요내용
  • 최근 경북지역의 산불발생으로 5월 1일 산사태위험지도 갱신하였으며, 갱신된 산사태위험도에 대하여 산사태전문가와 공간정보 전문가를 투입해서 개선이 필요한 부분을 점검

4) 산사태 지도서비스 개선사항

  • 현재 서비스되고 있는 산사태 지도서비스를 점검한 결과 대민서비스를 위한 지도서비스의 가독성과 속도 측면에서 개선이 필요함

  • 산사태 예보발령 위치를 확인하기 위해 행정구역을 선택하면 행정구역이 지도의 중심에 위치하지 않음(울릉군 => 독도가 중심)

  • (현상)산사태위험도 지도화면에서 위험도가 낮은 격자는 벡터데이터 형태의 파란색 색상으로 표시되어 나타나고 그 위에 행정구역명이 표시되나 파란색 색상에 한글이 가려져 가독성이 좋지 않음

    • (개선1)텍스트와 배경의 대비 색상 적용
    • (개선2)격자 색상의 투명도 조정
    • (개선3)텍스트 레이어를 가장 위에서 랜더링하도록 레이어 순서 조정

  • (현상)산사태위험지도 서비스에서 지도가 어느정도 축소된 상태에서는 표시되지만, 확대했을 때는 지도가 랜더링되다가 멈춘 상태로 표시되는 현상이 존재함

    • (원인1)지도 확대시에는 고해상도의 더 많은 타일이미지가 필요해지는데, 네트워크 지연이나 서버의 타일 생성/전송 속도 지연 등 병목현상이 발생하면 타일이 모두 로드되지 않아 지도가 멈춘 것처럼 표시됨.
    • (원인2) 타일 캐싱 미흡 : 지도 축소시에는 저해상도 타일을 사용하므로 빠르게 표시되나, 확대시에는 고해상도 타일을 서버에서 추가로 받아와서 표시해주어야 하나 이 과정에서 캐시가 잘 작동하지 않거나 캐시 미스로 인해서 로딩 지연 현상
    • 1차년도에 was서버 자원과 메모리 증설이 이루어진 것으로 보아, 타일캐싱문제일 것으로 판단
    • (개선) 타일 캐싱 최적화 하거나 서버의 타일 생성과 전송이 병목이면 자주 쓰는 영역은 미리 랜더링 하는 등 서버처리 능력 향상 필요

  • 산악 AWS : 산악지역에 설치된 자동기상관측장비 Automatic Weather System

  • 웹취약점은 정적테스트와 동적테스트로 구분할 수 있음,

  • 정적테스트는 소스코드상에 보안취약점이 존재하는지를 행얀부 보안취약점 49개 항목 대상으로 점검함. CC인증을 득한 도구를 활용하며 제안사는 CodeRay 툴을 이용함.

  • 동적테스트는 실제 실행환경에서 동적 웹 취약점을 점검.

    • CSRF(Cross-Site Request Forgery) : 인증된 사용자의 권한으로 공격자가 의도한 요청 실행하게 하는 취약점
    • 인증 우회 : URL 직접 접근, 쿠기 조작으로 인증 생략하는 취약점
    • 세션 하이재킹

  • OWASP ZAP, Burp Suite 등의 도구를 활용해서 점검
